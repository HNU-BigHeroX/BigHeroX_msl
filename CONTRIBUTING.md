# 参与贡献

感谢你对 BigHeroX MSL 项目的关注！我们欢迎任何有助于改进代码、文档和工具的贡献。

## 如何贡献

### 报告问题

如果你发现了 bug、文档错误或改进建议，请通过 [Issue](../../issues) 提交：

1. 使用清晰的标题描述问题。
2. 提供足够的环境信息（操作系统、编译器版本、硬件配置等）。
3. 描述复现步骤，并附上错误日志或截图。
4. 如果是代码问题，请指出涉及的文件和函数。

### 提交代码

1. **Fork 本仓库** 到你的账号下。
2. **创建分支**：从 `master` 分支创建功能分支，命名建议：
   - `feature/xxx` — 新功能
   - `fix/xxx` — 修复 bug
   - `docs/xxx` — 文档改进
3. **修改代码**：
   - 保持与现有代码风格一致，建议运行 `clang-format` 格式化 C++ 代码。
   - 添加必要的注释，关键逻辑保留中文或英文说明。
   - 如果修改涉及多模块，请在提交信息中说明影响范围。
4. **提交 Commit**：
   - 使用清晰的中文或英文提交信息。
   - 格式建议：`[模块] 简要描述`，例如 `[GoalKeeper] 修复守门员定位漂移问题`。
5. **发起 Pull Request**：
   - 描述改动内容和目的。
   - 关联相关的 Issue（如有）。
   - 等待维护者审阅和合并。

### 改进文档

文档贡献同样重要：

- 修正 `README.md`、`manual/` 中的错误或过时的描述。
- 补充工具使用说明、配置步骤或截图。
- 翻译或完善中英文文档。

## 代码规范

- **格式化**：C++ 代码请使用仓库根目录的 `.clang-format` 进行格式化。
- **命名**：尽量与现有代码保持一致。类名使用大驼峰，函数和变量使用小驼峰或下划线分隔。
- **注释**：复杂算法、比赛策略逻辑、硬件配置参数请保留注释，方便后续维护者理解。
- **兼容性**：本代码基于 Windows MFC 工程，提交前请确认能在兼容环境下编译通过。

## 沟通方式

- 一般性讨论：使用 [Discussions](../../discussions) 或相关 Issue。
- 紧急问题或安全相关：请直接联系维护者。

## 许可

通过向本仓库提交代码，你同意将你的贡献置于与项目相同的开源许可之下。详见 [LICENSE](LICENSE)。
